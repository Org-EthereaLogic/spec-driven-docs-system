# End-User Documentation

This directory contains documentation for **users** of the Spec-Driven Docs System framework.

## Audience

- Technical writers using the framework
- Developers implementing documentation workflows
- Team leads configuring documentation pipelines

## What Belongs Here

- **User guides:** Comprehensive usage guides for framework features
- **Tutorials:** Step-by-step tutorials for common workflows
- **Command reference:** Detailed reference for all slash commands
- **Examples:** Real-world examples and recipes
- **FAQ:** Frequently asked questions and troubleshooting

## What Does NOT Belong Here

- Generated technical documentation (use `spec_driven_docs/`)
- Framework configuration files (use `.claude/docs/`)
- Document specifications (use `specs/docs/`)
- Developer/project documentation (use `docs/`)

## Current Documentation

- [User Guide](User-Guide/User-Guide.md) - Comprehensive guide to all framework features

## Contributing

When adding end-user documentation:

1. Follow the existing style and structure
2. Include practical examples
3. Target the appropriate audience level
4. Cross-reference related documentation
